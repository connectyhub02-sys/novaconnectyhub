## Intentionally blocked provider paths

- `/instance/create` (POST): Use POST /instances. Raw provider creation requires admintoken and bypasses ConnectyHub ownership, billing and webhook setup.
- `/instance/all` (GET): Use GET /instances or admin-only internal inventory. Raw provider listing exposes instances across tenants.
- `/instance/updateAdminFields` (POST): Provider admin metadata is platform-owned and must not be writable by customer API keys.
- `/globalwebhook` (GET, POST): ConnectyHub exposes tenant-scoped /webhooks instead of provider-global webhooks.
- `/globalwebhook/errors` (GET): Use /webhooks/deliveries for tenant-scoped delivery diagnostics.
- `/admin/restart` (POST): Provider process control is platform-only.
- `/admin/token/rotate` (POST): Provider admin-token rotation is platform-only.
